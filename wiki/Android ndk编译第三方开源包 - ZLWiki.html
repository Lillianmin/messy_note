[编辑] 以编译libcurl为例,使用Android ndk编译arm的库

[编辑] 设置环境变量，使用ndk的prebuilt工具链 

export NDK_HOME=/media/mll/ylz_ccache/mll/dev/android-ndk
export PATH=$NDK_HOME:$PATH
export NDK_TOOLCHAIN_HOME=$NDK_HOME/toolchains
export ARMEABI_BIN=$NDK_TOOLCHAIN_HOME/arm-linux-androideabi-4.6/prebuilt/linux-x86_64/bin
export ARMEABI_LIB=$NDK_TOOLCHAIN_HOME/arm-linux-androideabi-4.6/prebuilt/linux-x86_64/lib:$LD_LIBRARY_PATH
export MIPSEL_BIN=$NDK_TOOLCHAIN_HOME/mipsel-linux-android-4.6/prebuilt/linux-x86_64/bin
export MIPSEL_LIB=$NDK_TOOLCHAIN_HOME/mipsel-linux-android-4.6/prebuilt/linux-x86_64/lib:$LD_LIBRARY_PATH
export X86_BIN=$NDK_TOOLCHAIN_HOME/x86-4.6/prebuilt/linux-x86_64/bin
export X86_LIB=$NDK_TOOLCHAIN_HOME/x86-4.6/prebuilt/linux-x86_64/lib:$LD_LIBRARY_PATH
export PATH=$ARMEABI_BIN:$MIPSEL_BIN:$X86_BIN:$PATH

[编辑] 编写编译脚本build_android.sh 

make distclean
export SYSROOT=$NDK_TOOLCHAIN_HOME/arm-linux-androideabi-4.6/sysroot
export CPPFLAGS="-I$SYSROOT/usr/include"
export LDFLAGS="-L$SYSROOT/usr/lib"
./configure --host=arm-linux-androideabi --prefix=$PWD/libs/armeabi
make
make install

[编辑] 编译中配置出现错误 

checking for arm-linux-androideabi-gcc... arm-linux-androideabi-gcc  --sysroot=/media/mll/ylz_ccache/mll/dev/android-ndk/platforms/android-19/arch-x86
checking whether the C compiler works... no
configure: error: in `/media/mll/debian_data/notebook/thirdpart/curl':
configure: error: C compiler cannot create executables
See `config.log' for more details

查看config.log,里面的提示有,如下内容。 

configure:4186: arm-linux-androideabi-gcc  --sysroot=/media/mll/ylz_ccache/mll/dev/android-ndk/platforms/android-19/arch-x86 -V >&5
arm-linux-androideabi-gcc: error: unrecognized option '-V'
arm-linux-androideabi-gcc: fatal error: no input files
compilation terminated.
configure:4197: $? = 1
configure:4186: arm-linux-androideabi-gcc  --sysroot=/media/mll/ylz_ccache/mll/dev/android-ndk/platforms/android-19/arch-x86 -qversion >&5
arm-linux-androideabi-gcc: error: unrecognized option '-qversion'
arm-linux-androideabi-gcc: fatal error: no input files
compilation terminated.

查找资料,不能直接使用$NDK_HOME/toolchains下的工具链，需要在NDK下提取出独立的工具链。

[编辑] NDK下提取出独立的工具链 

cd $NDK_HOME/build/tools
./make-standalone-toolchain.sh --ndk-dir=$NDK_HOME --system=linux-x86_64 --toolchain=arm-linux-androideabi-4.6
./make-standalone-toolchain.sh --ndk-dir=$NDK_HOME --system=linux-x86_64 --toolchain=x86-4.6
./make-standalone-toolchain.sh --ndk-dir=$NDK_HOME --system=linux-x86_64 --toolchain=mipsel-linux-android-4.6

在/tmp/ndk-mll/下生成了arm-linux-androideabi-4.6.tar.bz2 mipsel-linux-android-4.6.tar.bz2 x86-4.6.tar.bz2 将上面三个文件解压放至自己的独立工具文件夹下，$NDK_HOEM/standalone-toolchains 

mkdir $NDK_HOME/standalone-toolchains
mv /tmp/ndk-mll/*.tar.bz2 ./
tar jxvf arm-linux-androideabi-4.6.tar.bz2
tar jxvf mipsel-linux-android-4.6.tar.bz2
tar jxvf x86-4.6.tar.bz2

[编辑] 重新设置环境变量 

export NDK_HOME=/media/mll/ylz_ccache/mll/dev/android-ndk
export PATH=$NDK_HOME:$PATH
export NDK_TOOLCHAIN_HOME=$NDK_HOME/standalone-toolchains
export ARMEABI_BIN=$NDK_TOOLCHAIN_HOME/arm-linux-androideabi-4.6/bin
export ARMEABI_LIB=$NDK_TOOLCHAIN_HOME/arm-linux-androideabi-4.6/lib:$LD_LIBRARY_PATH
export MIPSEL_BIN=$NDK_TOOLCHAIN_HOME/mipsel-linux-android-4.6/bin
export MIPSEL_LIB=$NDK_TOOLCHAIN_HOME/mipsel-linux-android-4.6/lib:$LD_LIBRARY_PATH
export X86_BIN=$NDK_TOOLCHAIN_HOME/x86-4.6/bin
export X86_LIB=$NDK_TOOLCHAIN_HOME/x86-4.6/lib:$LD_LIBRARY_PATH
export PATH=$ARMEABI_BIN:$MIPSEL_BIN:$X86_BIN:$PATH

重新执行build_android.sh，至此编译libcurl的arm版本成功

[编辑] 以类似方法编译 sqlite3时问题

[编辑] configure不识别arm-linux-androideabi 

checking build system type... Invalid configuration `arm-linux-androideabi': system `androideabi' not recognized
configure: error: /bin/bash ./config.sub arm-linux-androideabi failed

查询资料，configure-does-not-recognize-androideabi 是因为sqlite3版本比较老，是2009-12-07版本，而config.sub与config.guess配置文件需要是2010-05-20以 后的版本才支持android，所以更新这两个文件，将系统的config.sub config.guess拷贝至此。 

sudo cp /usr/share/misc/config.{sub,guess} .
sudo chown mll:mll config.sub config.guess

[编辑] libsqlite3.so: error: undefined reference to 'fdatasync'

说是要sqlite3库的一个BUG,需要更新版本才能解决。下载 sqlite-autoconf-3081002.tar.gz版本，编译无问题
