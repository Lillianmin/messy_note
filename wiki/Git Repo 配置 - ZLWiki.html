[编辑] 创建基础repo 服务器和客户端

[编辑] repo 服务器 

cd $SERVER
mkdir manifests
cd manifests
git init
# create default.xml

创建简单 default.xml 文档 

<?xml version="1.0" encoding="UTF-8"?>
<manifest>
	<remote name="init"
		fetch="." />
	<default revision="master"
		remote="init" />
</manifest>
mv .git ../manifests.git
ln -s ../manifests.git .git
git add -A  #current dir is manifests. use to commit default.xml
git commit -a -m "repo init"

[编辑] repo 客户端 

mkdir client
cd client
repo init -u git@192.168.4.253:/srv/gitdir/android-apps-repo/.repo/manifests.git
or 
repo init --repo-url=git@192.168.4.253:/srv/gitdir/android-apps-repo/git-repo.git -u git@192.168.4.253:/srv/gitdir/android-apps-repo/manifests.git
repo sync

到目前为止，已经创建了一个最基本的repo服务器和客户端了。接下来进入我们自己的项目repo管理。 repo 管理项目可以通过服务器的分支和服务器上不同的配置文件两种途径来设置。

[编辑] 使用不同配置文件管理不同项目

这种管理不通过branch来管理项目，而是通过不同的配置文件来管理项目。目前ebookbag项目使用该管理方式。

[编辑] repo 服务器 

cd /srv/gitdir/android-apps-repo/.repo/manifests/
git checkout master

创建 ebookbag.xml 

<?xml version="1.0" encoding="utf-8"?>
<manifest>
        <remote name="ebookbag" fetch="./" />
        <default revision="master" remote="ebookbag" />
        <project name="ebookbag/network" path="ebookbag/network" />
        <project name="ebookbag/networktest" path="ebookbag/networktest" />
        <project name="ebookbag/filedialog" path="ebookbag/filedialog" />
        <project name="ebookbag/ebookviewer" path="ebookbag/ebookviewer" />
        <project name="ebookbag/ebagcommon" path="ebookbag/ebagcommon" />
        <project name="ebookbag/ebagdesktop" path="ebookbag/ebagdesktop" />
        <project name="ebookbag/ebagexercise" path="ebookbag/ebagexercise" />
        <project name="ebookbag/handwrittenrecognition" path="ebookbag/handwrittenrecognition" />
</manifest>
git commit -a

[编辑] repo 客户端下载ebookbag项目 

repo init  -u git@192.168.4.253:/srv/gitdir/android-apps-repo/manifests.git -m ebookbag.xml

即将所有的 ebookbag 项目的工程拉下来。


[编辑] 客户端使用

[编辑] 获取repo

[编辑] 获取所有工程 

repo init  -u git@192.168.4.253:/srv/gitdir/android-apps-repo/manifests.git -b master

cd .repo/manifests

  1. 执行下列操作让本地的master分支与远程的master分支对应，让本地的default分支与远程的default分支对应。
  2. 如果不执行。默认是本地default分支对应远程master分支。
  3. 要在repo sync前执行下列操作，否则无法checkout master。git branch master新建的也不是与远程对应的。 
git checkout master
git branch -d default
git checkout default

如果已经先repo sync下来了。并且default对应远程master分枝了，本地修改default.xml无法提交。则采用以下方法解决： 

git checkout -b master
git pull 
git branch --set-upstream master origin/master
git pull
git branch -d default
git checkout default
git pull

[编辑] 获取 ebookbag 项目工程 

repo init  -u git@192.168.4.253:/srv/gitdir/android-apps-repo/manifests.git -m ebookbag.xml


[编辑] 同步所有工程 

repo sync

使用这个命令的时候需要注意本地是否都已经提交，如果没有提交，会将未提交文件覆盖。

[编辑] 使用 master 分支 

repo start master --all

第一次下载工程文件后，最好将所有项目都切换到 master 分支。

之后，每个工程都可以按照git命令来操作。

[编辑] 新建工程 

ssh mll@192.168.4.253
cd /srv/gitdir/android-apps-repo/ebookbag/
 
mkdir test_project
cd test_project
git init --bare 
 
cd /srv/gitdir/android-apps-repo/.repo/manifests
vi ebookbag.xml
添加 <project name="test_project" path="project" />
 
git commit -a

客户端 

repo sync


[编辑] 往远程repo添加工程：

本地： 在本地新建安卓工程ebagdraw 

cd /mll/workspace/ebagdraw
git init
git add -A
git commit -a -m "create ebagdraw"

远程： 

cd /srv/gitdir/android-apps-repo/ebookbag
git clone mll@192.168.4.115:/home/mll/workspace/ebagdraw ebagdraw.git --bare
cd /srv/gitdir/android-apps-repo/.repo/manifests
git checkout master


说明：只有切换到master提交的default.xml，通常情况下，在客户端 repo sync 时才会同步下来，否则同步不下来。

说明：repo init下来，默认对应的本地分支是default。 所以在修改服务器上default.xml等文件时，应该切换到master分支下面。否则，默认情况下本地repo sync时无法同步下来。

具体情况可以参见 获取所有工程 后面的说明。 如果本地分枝和远程分枝一一对应，修改defaults.xml等文件的工作也可以在本地的repo_dir/.repo/manifests下面修改，并向远程提交。 修改defaults.xml ebookbag.xml 添加 <project name="ebookbag/ebagdraw" path="ebookbag/ebagdraw" /> 说明：在提交之前确保ebookbag/ebagdraw的git项目存在，否则在repo sync时，会提示ebookbag/ebagdraw does not appear to be a git repository 记得在新项目中添加 .gitignore 

git add -A
git commit

本地： 

cd /home/mll/RepoProject
repo sync



这个时候可能会有输入口令问题，建议按照以下方法解决: 在自己本机上，比如：192.168.4.115 执行： 

cd ~/.ssh
ssh-keygen

将生成的 id_rsa.pub 拷贝到服务器192.168.4.253上 

scp id_rsa.pub root@192.168.4.253:/home/git/.ssh
ssh root@192.168.4.253
cd /home/git/.ssh
cat id_rsa.pub >> authorized_keys

这样今后提交都可以不需要输入口令。

这样设置之后登录时仍旧提示“Agent admitted failure to sign using the key.” 则在自己本机上面执行“ssh-add ~/.ssh/id_rsa”后，再次登录不用输入密码

[编辑] 添加目录

如果使用过程中需要给工程分开目录，可以参考 ebookbag 的创建方法 在 192.168.4.253:/srv/gitdir/android-apps-repo 下创建一个目录。 然后在该目录下添加不同的git库。

以 ebookbag 为例： 

cd .repo/manifests/
vi default.xml

添加 

        <project name="ebookbag/network" path="ebookbag/network" />
        <project name="ebookbag/networktest" path="ebookbag/networktest" />


git commit -a

客户端执行 

repo sync
