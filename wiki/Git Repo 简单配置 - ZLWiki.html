[编辑] 简便创建repo服务器方法

[编辑] 创建manifests.git仓库

$ mkdir -p /path/to/repo/.repo
$ cd /path/to/repo/.repo
$ mkdir manifests.git; 
$ cd manifests.git
$ git init --bare

[编辑] 在.repo目录创建manifests.git的克隆 

$ cd /path/to/repo/.repo
$ git clone manifests.git manifests
$ cd manifests

[编辑] 添加编辑default.xml

<?xml version="1.0" encoding="UTF-8"?>
<manifest>
        <remote name="android-apps" fetch="./" />
        <default revision="master" remote="android-apps" />
</manifest>
$ git add default.xml
$ git commit -a 
$ git push

[编辑] 创建链接

$ cd /path/to/repo/
$ ln -s .repo/manifests.git manifests.git

到这里 repo 服务器基本上架好了。

[编辑] 客户端使用

$ repo init --repo-url=git@192.168.4.253:/srv/gitdir/android-apps-repo/git-repo.git -u user@ip:/path/to/repo/manifests.git -b master
$ repo sync
$ repo start master --all

[编辑] 注意事项

[编辑] 服务器端

如果出现：error: revision master in PROJECT not found，需要查看该git项目是否为空项目。

[编辑] 客户端

添加git项目只需要编辑.repo/manifests/default.xml。但是需要提交到master分支才行。 

$ git branch master origin/header/master 就可以创建master分支，
$ git checkout master
然后编辑default.xml。
$ git commit -a; git push

基本上这就可以大功告成了。
