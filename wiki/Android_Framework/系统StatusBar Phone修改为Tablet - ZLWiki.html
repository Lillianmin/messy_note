系统StatusBar Phone修改为Tablet

跳转至： 导航、 搜索

status bar在 SystemUI中，SystemUI路径：

frameworks/base/packages/SystemUI/src/com/android/systemui

SystemUIService在frameworks/base/services/java/com/android/server/SystemServer.java中启动

frameworks/base/packages/SystemUI/src/com/android/systemui/SystemUIService中 

SERVICES[0] = wm.hasSystemNavBar()
                    ? R.string.config_systemBarComponent
                    : R.string.config_statusBarComponent;

由wm.hasSystemNavBar()来决定是Phone 还是 Tablet。

frameworks/base/policy/src/com/android/internal/policy/impl/PhoneWindowManager.java中 

 if (shortSizeDp < 600) {
            // 0-599dp: "phone" UI with a separate status & navigation bar
            mHasSystemNavBar = false;
            mNavigationBarCanMove = true;
        } else {
            // 720dp: "tablet" UI with a single combined status & navigation bar
            mHasSystemNavBar = true;
            mNavigationBarCanMove = false;
        }

屏幕小边的dp<600，则是Phone否则是Tablet. 我们Eink 6寸是571dp，将这里600改成500.

屏幕小边dp计算：

以6寸eink为例

1024*758

dpi = √(758²+1024²)/对角线英寸

dpi = 212

dp=px*160/dpi

w:571dp

另外，SystemUI res下面针对Tablet的有一部分写死为drawable-sw600dp* layout-sw600dp* values-sw600dp*，针对500的，也需要做一份对应的drawable/layout/values。初步将对应的600的拷贝成一份 500的可以解决。

如果这里资源没有准备对应500的，SystemUI起来的时候会报找不到资源的错误。

frameworks/base/packages/SystemUI/src/com/android/systemui/statusbar/SignalClusterView.java

apply()中，屏蔽 mWifiActivity.setImageResource(myWifiActivityId);

[编辑] SystemUI statusbar 修改

  1. disable statusbar expand
frameworks/base/packages/SystemUI/src/com/android/systemui /statusbar/tablet/TabletStatusBar.java中NotificationTriggerTouchListener onTouch中return true。但是这样Notification的动作也没法处理了。
  1. only disable statusbar expand settings, allow notification
替代方案：去掉SystemUI NotificationPannel中的Setting相关，NotificationPanelTitle中只保留时间相关与清除Notification的按扭，隐藏进入设置的按扭。

但是这样，USB的Notification点击后还是会进入到系统设置，删除Settings.apk。若同时也将 SettignsProvider.apk删除，系统无法启动。但是这样的话，Notification只是会弹出一下，在 NotificationPannel中没有。

查看TabletStatusBar源码， 

(isDeviceProvisioned()||showNotificationEvenIfUnprovisioned(sbn)){
    toShow.add(xxx);
}

表示当系统是unprovisioned时， 只显示系统的Notification。

isDeviceProvisioned()来自Settings.Secure.DEVICE_PROVISIONED，默认为 false（frameworks/base/packages/SettingsProvider/res/values/defaults.xml name="def_deivce_provisioned"）。由于Settings.apk去掉了，故isDeviceProvisioned为 false。修改showNotificationEvenIfUnprovisioned(sbn)，返回true,让 DEVICE_PROVISIONED为false的情况下非系统的Notification也能添加至NotificationPanel中。这样 NotificationPanel的clearButton不会显示： 

setAreThereNotifications(){
    if(mNotificationPanel != null){
        mNotificationPanel.setClearable(isDeviceProvisioned()&&mNotificationData.hasClearableItem());
        }
}

因为在isDeviceProvisioned（）为false时，不会设置NotificationPanel中的Notificatio为clearable。
