让system应用中可以执行su

来自ZLWiki
跳转至： 导航、 搜索

问题来源

添加EinkControl至系统应用中，有system权限，也无法echo 数据至/sys/devices/platform/jz-fb.1/epd_mode中。 然后偿试使用su来执行echo命令，输出错误：su: uid 1000 not allowed to su。


默认只有root和shell用户可以使用su，由于在system应用中需要使用su，所以加上AID_SYSTEM也可以使用su。 在su.c中修改 system/extras/su/su.c 

 myuid = getuid();
 if (myuid != AID_ROOT && myuid != AID_SHELL) {
         fprintf(stderr,"su: uid %d not allowed to su/n", myuid);
         return 1;
 }
 myuid = getuid();
 if (myuid != AID_ROOT && myuid != AID_SHELL && myuid != AID_SYSTEM) {
         fprintf(stderr,"su: uid %d not allowed to su/n", myuid);
         return 1;
 }

添加AID_SYSTEM。各AID常量在下面的文件中。 system/core/include/private/android_filesystem_config.h
